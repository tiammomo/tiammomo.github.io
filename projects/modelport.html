Case Study

ModelPort

一个面向 Claude Code 和 VS Code Claude 的本地模型网关：保持编辑器工作流不变，同时把多模型接入、协议转换、路由策略、鉴权和用量观测收敛到一个自托管入口。

查看仓库 更多项目
Problem

多模型能力很多，但开发者入口应该稳定

不同模型提供商有不同协议、模型命名、鉴权方式和稳定性表现。ModelPort 的核心价值是让开发者继续使用熟悉的 Claude 客户端，同时在本地网关里处理 Provider Routing、协议适配、预算和观测。

My Role

从协议边界到可运行控制面板

这个项目体现的是工程基础设施能力：入口 API、上游转换、流式响应、配置校验、用量统计、Dashboard、Docker Compose、systemd 和项目文档一起构成可验证交付。

核心能力

  • Anthropic-compatible `/v1/messages` 入口。
  • Provider Routing、模型别名、模型前缀和未知模型透传。
  • 本地 Token 鉴权、API Keys、团队/项目、预算和 Allowlist。
  • 请求日志、Token 用量、延迟、成本估算和 Prometheus 指标。
  • Docker Compose、配置校验、doctor、provider matrix 和验收文档。

展示价值

对外展示时，ModelPort 是最能说明“AI 产品工程”能力的项目：它不是单个 Prompt Demo，而是把模型接入、工程边界、安全策略、可观测性和开发者体验合在一起的工具系统。
