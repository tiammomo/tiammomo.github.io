Case Study

moyuan-travel-agent

一个旅行规划 Agent 原型，围绕用户偏好、预算约束、目的地比较和路线生成做决策辅助，展示 AI 助手从对话走向产品流程的路径。

查看仓库 更多项目
◇
Problem

旅行规划的难点在于约束很多，答案不止一个

用户常常同时关心预算、时间、城市偏好、交通方式、路线顺序和体验节奏。这个项目把这些因素组织成可比较的规划过程，而不是只给出一段静态行程。

◆
My Role

把 Agent 决策做成可理解的产品流程

项目重点是产品设计和 Agent 流程结合：收集偏好、生成候选方案、比较路线、解释权衡，最后输出用户能直接调整和使用的行程结果。

◇
Capabilities

核心能力

  • 偏好、预算和时间约束的结构化表达。
  • 多候选路线比较，而不是单一路线输出。
  • 把行程生成、解释和调整放进同一体验链路。
  • 适合作为垂直领域 Agent 产品化的轻量样板。
◆
Value

展示价值

Travel Agent 项目能展示你对"Agent 不只是调用模型"的理解：它需要信息收集、约束建模、结果解释和界面反馈一起工作。
