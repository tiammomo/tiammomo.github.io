这里用来记录 AI 产品工程里的设计取舍：模型网关、Agent 工作流、数据工具、产品原型和开源项目交付。它会随着项目继续增长。

LLM Gateway · Architecture

如何设计一个本地 LLM Gateway

从编辑器入口、协议兼容、Provider Routing、鉴权、预算和观测开始，拆解 ModelPort 这类工具应该守住的工程边界。

  • 入口 API 要稳定，Provider 可以替换。
  • 路由、预算、日志和指标要一起设计。
  • 个人本地工具和公开 SaaS 网关是两种边界。
Agent Product · Workflow

Agent 产品和普通 Chatbot 的区别

Agent 不是更长的对话，而是目标、约束、工具、状态和可回放结果组成的工作流。产品界面需要承接这些中间状态。

  • 把用户目标转成结构化任务。
  • 让候选方案、权衡和修改过程可见。
  • 把结果沉淀成下一次可以复用的资产。
AI Product · Prototype

AI 工具如何从 Demo 走向产品

一个有趣 Demo 往往只证明模型能做事；产品原型还需要处理失败路径、数据入口、用户反馈、权限边界和迭代节奏。

  • 先做能验证的产品切片。
  • 为失败、空状态和人工修正留位置。
  • 用日志和反馈回路驱动下一轮迭代。
Quant Workspace · Data

量化投研工作台的产品结构

投研场景里，Agent 的价值不只是回答问题，而是帮助整理数据、推进分析、记录假设、比较结果并支持复盘。

  • 策略假设、数据处理和结果展示要在同一链路。
  • 图表和 notebook 需要服务复盘，而不是只服务展示。
  • 让 Agent 产出成为工作区资产。
